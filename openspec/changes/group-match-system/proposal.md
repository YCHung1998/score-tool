## Why

現有工具以「個人選手得分」為核心：每場輸入每位選手分數，依勝場數排名。但實際使用情境是「小組分組對抗賽」——玩家被編入小組（3 人），小組再組合成 6 人隊伍對打，最終以小組為單位計算排名。兩者的資料模型根本不同，現有工具無法直接支援此賽制。

**實際賽制範例（6/19）：**

| 小組 | 成員 |
|------|------|
| A | 昱佑、歐易、消麵 |
| B | 翊誠、唯鈞、庭伊 |
| C | 志平、芳伶、秀蘭 |
| D | 小高、裕翔、姵萱 |
| X | 昀、菜菜、謝汶、偉、周周、哲綸（6人獨立隊） |

第一輪場次：AC vs X、AC vs BD、X vs BD
第二輪場次：AD vs BC、X vs BC、X vs AD
第三輪場次（重新分組 E/F/G/H）：EG vs FH、EH vs X、X vs FG

排名計算：以**小組**為單位，統計勝場數與分數差，得出最終名次。

## What Changes

- 移除個人選手逐場得分的計分模式
- 改為「小組」+「賽程場次」的競賽模式：
  - **小組（小組）**：命名小組，登記成員，作為排名基本單位
  - **場次（Match）**：指定兩側隊伍各由哪些小組組成，記錄兩側比賽分數
  - **輪次（Round）**：場次可附加輪次標籤（第一輪 / 第二輪…），方便管理
- 排名引擎改為：依小組跨所有場次的勝場數與分數差排序
- 重組後（如 EFGH）的小組視為新的獨立小組定義
- 保留選手名單功能（成員顯示用）
- 保留匯出 / 匯入功能（新格式）

## Capabilities

### New Capabilities

- `group-roster`: 定義小組（名稱 + 成員），作為排名與賽程的基本單位
- `match-schedule`: 建立場次，指定兩側各由哪些小組組成隊伍
- `match-scoring`: 為每場次輸入兩側分數，系統自動判定勝負
- `group-rankings`: 依勝場數 → 分數差排名各小組

### Modified Capabilities

- `score-history`: 場次紀錄取代逐輪個人分數紀錄
- `ui-theme`: 無破壞性更動，新 UI 沿用現有 Apple 色調與樣式

## Impact

- `index.html`：完整替換 JS 狀態模型、UI 邏輯、四個主要面板的 HTML
- 匯出格式：全新 TXT 格式（舊格式不相容）
- 排名引擎：全新實作
