## 1. Git Setup

- [x] 1.1 Run `git init` in the project root (`/Users/eason.hung/Documents/score_tool`)
- [x] 1.2 Create `.gitignore` with entries for `.DS_Store`, `*.swp`, and other OS/editor noise
- [x] 1.3 Stage all files (`git add .`) and create the initial commit

## 2. GitHub Repository

- [x] 2.1 Create a new GitHub repository via `gh repo create` (choose name, e.g. `score-tool`, and visibility)
- [x] 2.2 Push the `main` branch to the new remote (`git push -u origin main`)

## 3. Enable GitHub Pages

- [x] 3.1 In the GitHub repo Settings → Pages, set Source to `Deploy from a branch`, branch `main`, folder `/ (root)`
- [x] 3.2 Wait for GitHub Pages to build (usually under 1 minute) and confirm the URL is shown in Settings → Pages

## 4. Verify

- [ ] 4.1 Open the published GitHub Pages URL in a browser and confirm the app loads
- [ ] 4.2 Test a core flow on the live site (add a player, enter a score, check rankings)
